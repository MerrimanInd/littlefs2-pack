Getting Started

ESP-Dashboard is an open-source IoT monitoring platform designed for ESP32 microcontrollers. It provides real-time sensor data visualization, device management, and alerting capabilities.

Prerequisites

Before you begin, make sure you have the following:

An ESP32 development board (ESP32-DevKitC or similar), a USB cable for programming, Arduino IDE or PlatformIO installed, Wi-Fi network access, and at least one supported sensor module.

Quick Start

# Clone the repository
git clone https://github.com/esp-dashboard/firmware.git
cd firmware

# Install dependencies
pio lib install

# Configure Wi-Fi credentials
cp config.example.h config.h
nano config.h

# Build and flash
pio run --target upload

# Monitor serial output
pio device monitor

After flashing, the ESP32 will connect to your Wi-Fi network and start broadcasting sensor data. The dashboard will be accessible at the device's IP address on port 80.

Hardware Setup

Supported Sensors

ESP-Dashboard supports a wide range of I2C, SPI, and analog sensors. The most commonly used sensors include the BME280 (temperature, humidity, pressure), BH1750 (light intensity), PIR motion sensors, and various gas sensors.

Wiring Diagram

Connect sensors to the ESP32 using the following pin mapping:

BME280 (I2C):
  SDA → GPIO 21
  SCL → GPIO 22
  VCC → 3.3V
  GND → GND

BH1750 (I2C):
  SDA → GPIO 21
  SCL → GPIO 22
  ADDR → GND (address 0x23)
  VCC → 3.3V
  GND → GND

PIR Sensor:
  OUT → GPIO 34
  VCC → 3.3V
  GND → GND

Power Considerations

For battery-powered deployments, ESP-Dashboard supports deep sleep modes with configurable wake intervals. The system consumes approximately 150mA during active transmission and can be reduced to under 10μA in deep sleep mode.

API Reference

REST API Endpoints

The ESP32 exposes a REST API for querying sensor data and managing device configuration.

GET /api/sensors
Returns current readings from all connected sensors.

Response:
{
    "temperature": 22.5,
    "humidity": 55.2,
    "pressure": 1013.4,
    "light": 640,
    "motion": false,
    "timestamp": 1700000000
}

GET /api/sensors/history?hours=24
Returns historical data for the specified time range.

GET /api/device/info
Returns device information including firmware version,
uptime, memory usage, and network status.

POST /api/device/config
Update device configuration parameters.

Body:
{
    "sample_interval": 300,
    "deep_sleep": true,
    "wifi_power": 8,
    "mqtt_enabled": true
}

POST /api/device/restart
Trigger a device restart.

GET /api/alerts
Returns recent alert history.

WebSocket API

For real-time data streaming, connect to the WebSocket endpoint:

ws://[device-ip]/ws

// Subscribe to sensor updates
{"action": "subscribe", "sensors": ["temperature", "humidity"]}

// Incoming data format
{"sensor": "temperature", "value": 22.5, "ts": 1700000000}

Configuration

Configuration File

ESP-Dashboard uses a header file for compile-time configuration and a JSON file for runtime settings. The compile-time configuration includes Wi-Fi credentials, MQTT broker settings, and pin assignments.

// config.h — Compile-time configuration
#define WIFI_SSID     "YourNetworkName"
#define WIFI_PASSWORD "YourPassword"
#define DEVICE_NAME   "esp32-001"
#define MQTT_BROKER   "192.168.1.50"
#define MQTT_PORT     1883
#define MQTT_USER     "dashboard"
#define MQTT_PASS     "secret"

// Sensor pins
#define I2C_SDA       21
#define I2C_SCL       22
#define PIR_PIN       34
#define LED_PIN       2

// Intervals (milliseconds)
#define SAMPLE_INTERVAL   5000
#define REPORT_INTERVAL   60000
#define DISPLAY_INTERVAL  1000

Runtime Configuration

Runtime settings can be adjusted via the web interface or API without reflashing. These are stored in SPIFFS/LittleFS on the ESP32's flash memory.

// /config.json — Runtime configuration
{
    "hostname": "esp-dashboard",
    "timezone": "UTC+0",
    "temp_unit": "C",
    "alert_thresholds": {
        "temp_high": 35.0,
        "temp_low": 5.0,
        "humidity_high": 80.0,
        "humidity_low": 20.0
    },
    "display": {
        "brightness": 80,
        "auto_off": 300,
        "theme": "dark"
    },
    "logging": {
        "level": "info",
        "serial": true,
        "remote": false
    }
}

Troubleshooting

Common Issues

Device won't connect to Wi-Fi: Verify your SSID and password in config.h. Ensure the network is 2.4GHz (ESP32 doesn't support 5GHz). Check that the router isn't blocking new devices. Try reducing Wi-Fi TX power in configuration.

Sensor readings are NaN or zero: Check wiring connections, especially I2C pull-up resistors (4.7kΩ on SDA and SCL). Verify sensor I2C address with an I2C scanner sketch. Some sensors require a warm-up period of 1-2 seconds after power-on.

Web interface not loading: Verify the SPIFFS/LittleFS filesystem is properly uploaded. Use pio run --target uploadfs to upload web assets. Check available flash space — the web interface requires approximately 2MB.

High memory usage: Monitor free heap with ESP.getFreeHeap(). Reduce JSON buffer sizes if running low. Consider using streaming JSON for large responses. Disable features you don't need (MQTT, OTA, etc.).
